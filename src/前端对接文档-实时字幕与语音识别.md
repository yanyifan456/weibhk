# 实时字幕与语音识别 - 前端对接文档

## 一、整体流程概览

```
┌──────────┐    HTTP POST     ┌──────────────────┐    Feign     ┌──────────────────┐
│          │ /video/record/start│                  │             │                  │
│  前端    │ ───────────────► │ medicineconsumer  │ ──────────► │ medicineprovider  │
│(医生/患者)│                  │   (API网关层)     │             │   (核心业务层)    │
│          │ ◄─────────────── │                  │ ◄────────── │                  │
└────┬─────┘   返回taskId等    └──────────────────┘   返回结果   └────────┬─────────┘
     │                                                                     │
     │                                                         启动TRTC录制+ASR识别
     │                                                         简繁转换/TMT翻译
     │                                                         推送字幕→infusionalarm
     │                                                                     │
     │  WebSocket (发送PCM音频)                                            │
     ├──────────────────────────► ┌──────────────┐ ──HTTP──► ┌───────────┤
     │  /ws/audio/{taskId}        │              │           │           │
     │                            │infusionalarm │           │  Provider  │
     │                            │ (WebSocket层) │ ◄─HTTP─── │  ASR处理   │

     │  /ws/subtitle/{roomId}/    └──────────────┘           └───────────┘
     │  {userId}
     ▼
  显示字幕
```

## 二、接口详情

### 1. 开始视频录制 + 启动语音识别

**请求**

```
POST /video/record/start
Content-Type: application/json
```

**请求体**

```json
{
  "roomId": "room_12345",
  "userId": "13800138000",
  "doctorId": "100",
  "orderId": "ORD001",
  "doctorSpeakLanguage": "yue-CN",
  "patientSpeakLanguage": "zh-CN",
  "patientOutputFormat": "simplified",
  "doctorOutputFormat": "traditional"
}
```

**参数说明**

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `roomId` | String | ✅ | - | TRTC房间号 |
| `userId` | String | ✅ | - | 用户手机号（后端自动加密查询真实userId） |
| `doctorId` | String | ✅ | - | 医生ID（数据库ID） |
| `orderId` | String | ❌ | - | 订单ID |
| `doctorSpeakLanguage` | String | ❌ | `"yue-CN"` | 医生说话语言 |
| `patientSpeakLanguage` | String | ❌ | `"zh-CN"` | 患者说话语言 |
| `patientOutputFormat` | String | ❌ | `"simplified"` | 患者看到医生字幕的翻译目标 |
| `doctorOutputFormat` | String | ❌ | `"traditional"` | 医生看到患者字幕的翻译目标 |

**语言枚举值**

| 值 | 含义 |
|----|------|
| `zh-CN` | 普通话 |
| `yue-CN` | 粤语 |
| `en` | 英语 |

**输出格式枚举值**

| 值 | 含义 | 说明 |
|----|------|------|
| `simplified` | 简体中文 | 输出简体中文字幕 |
| `traditional` | 繁体中文 | 输出繁体中文字幕 |
| `en` | 英语 | 输出英语字幕（跨语言翻译） |
| `none` | 不转换 | 保持ASR原始输出 |

**响应**

```json
{
  "code": 200,
  "data": {
    "success": true,
    "message": "录制记录创建成功",
    "recordId": 1,
    "doctorTaskId": "room_12345_doctor_100_1717300000000",
    "userTaskId": "room_12345_13800138000_1717300000001",
    "audioWsUrl": "ws://192.168.100.14:8089/ws/audio/"
  }
}
```

**响应字段**

| 字段 | 类型 | 说明 |
|------|------|------|
| `recordId` | Long | 录制记录ID（停止录制时使用） |
| `doctorTaskId` | String | 医生端语音识别任务ID（连接音频WebSocket时使用） |
| `userTaskId` | String | 患者端语音识别任务ID（连接音频WebSocket时使用） |
| `audioWsUrl` | String | 音频流WebSocket地址前缀 |

---

### 2. 停止视频录制

**请求**

```
POST /video/record/stop/{recordId}
```

**响应**

```json
{
  "code": 200,
  "data": {
    "success": true,
    "message": "录制已停止"
  }
}
```

> 关闭音频流WebSocket连接时，后端会自动停止对应的语音识别任务，无需额外调用停止接口。

## 三、WebSocket连接

### 1. 音频流WebSocket（发送PCM音频）

**连接地址**

```
ws://{audioWsUrl}{taskId}
```

示例：
- 医生端：`ws://192.168.100.14:8089/ws/audio/room_12345_doctor_100_1717300000000`
- 患者端：`ws://192.168.100.14:8089/ws/audio/room_12345_13800138000_1717300000001`

**发送格式**

- 二进制帧（Binary Frame）
- PCM格式：16kHz采样率、16bit位深、单声道
- 建议每次发送 40ms 的音频数据（640字节）

**前端代码示例**

```javascript
// 1. 调用开始录制接口获取taskId和audioWsUrl
const startResult = await fetch('/video/record/start', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    roomId: 'room_12345',
    userId: '13800138000',
    doctorId: '100',
    doctorSpeakLanguage: 'yue-CN',
    patientSpeakLanguage: 'zh-CN',
    patientOutputFormat: 'simplified',
    doctorOutputFormat: 'traditional'
  })
}).then(r => r.json());

const { doctorTaskId, userTaskId, audioWsUrl } = startResult.data;

// 2. 根据角色选择对应的taskId
const myTaskId = isDoctor ? doctorTaskId : userTaskId;

// 3. 连接音频流WebSocket
const audioWs = new WebSocket(audioWsUrl + myTaskId);
audioWs.binaryType = 'arraybuffer';

// 4. 从TRTC获取本地音频流，转PCM后发送
// 具体音频采集方式取决于前端TRTC SDK的使用方式
audioWs.onopen = () => {
  console.log('音频流WebSocket已连接');
};

// 5. 关闭连接（停止录制或离开房间时）
function stopAudioStream() {
  if (audioWs.readyState === WebSocket.OPEN) {
    audioWs.close();
  }
}
```

### 2. 字幕WebSocket（接收实时字幕）

**连接地址**

```
ws://{infusionalarmHost}/ws/subtitle/{roomId}/{userId}
```

示例：
- 医生端：`ws://192.168.100.14:8089/ws/subtitle/room_12345/doctor_100`
- 患者端：`ws://192.168.100.14:8089/ws/subtitle/room_12345/13800138000`

> **注意**：`userId` 需要与开始录制时使用的userId一致。医生端为 `doctor_{doctorId}`，患者端为手机号（后端加密后的真实userId会自动匹配）。

**接收消息格式**

服务端推送的每条消息均为JSON字符串：

```json
{
  "type": "text",
  "speakerId": "doctor_100",
  "speakerRole": "doctor",
  "originalText": "你好嗎",
  "convertedText": "你好吗",
  "language": "yue-CN",
  "targetLanguage": "zh-CN",
  "roomId": "room_12345",
  "isFinal": true,
  "timestamp": 1717300000000
}
```

**字段说明**

| 字段 | 类型 | 说明 |
|------|------|------|
| `type` | String | 消息类型，固定为 `"text"` |
| `speakerId` | String | 说话人ID |
| `speakerRole` | String | 说话人角色：`"doctor"` / `"user"` |
| `originalText` | String | ASR原始识别文本 |
| `convertedText` | String | 转换/翻译后的文本（**前端应显示此字段**） |
| `language` | String | 说话人原始语言：`"zh-CN"` / `"yue-CN"` / `"en"` |
| `targetLanguage` | String | 转换目标语言：`"zh-CN"` / `"zh-TW"` / `"en"` |
| `roomId` | String | 房间ID |
| `isFinal` | Boolean | `true`=最终结果（一句说完），`false`=中间结果（正在说） |
| `timestamp` | Long | 消息时间戳（毫秒） |

**前端代码示例**

```javascript
const roomId = 'room_12345';
const userId = isDoctor ? 'doctor_100' : '13800138000';

const subtitleWs = new WebSocket(
  `ws://192.168.100.14:8089/ws/subtitle/${roomId}/${userId}`
);

subtitleWs.onmessage = (event) => {
  const data = JSON.parse(event.data);

  if (data.type === 'text') {
    // 使用 convertedText 显示字幕（已翻译为目标语言）
    // 使用 isFinal 区分中间结果和最终结果
    if (data.isFinal) {
      // 最终结果：固定显示
      appendFinalSubtitle(data.speakerRole, data.convertedText);
    } else {
      // 中间结果：实时更新（打字机效果）
      updateInterimSubtitle(data.speakerRole, data.convertedText);
    }
  }
};

subtitleWs.onclose = () => {
  console.log('字幕WebSocket已断开');
};
```

## 四、转换规则矩阵

### 医生说话 → 患者看到（由 `patientOutputFormat` 控制）

| 医生说话语言 | patientOutputFormat | ASR识别结果 | 转换方式 | 患者看到 | targetLanguage |
|-------------|---------------------|------------|---------|---------|----------------|
| `yue-CN` 粤语 | `simplified` | 繁体中文 | 繁→简(opencc4j) | 简体中文 | `zh-CN` |
| `yue-CN` 粤语 | `traditional` | 繁体中文 | 不转换 | 繁体中文 | `zh-TW` |
| `yue-CN` 粤语 | `en` | 繁体中文 | TMT翻译 | 英语 | `en` |
| `zh-CN` 普通话 | `simplified` | 简体中文 | 不转换 | 简体中文 | `zh-CN` |
| `zh-CN` 普通话 | `traditional` | 简体中文 | 简→繁(opencc4j) | 繁体中文 | `zh-TW` |
| `zh-CN` 普通话 | `en` | 简体中文 | TMT翻译 | 英语 | `en` |
| `en` 英语 | `simplified` | 英语 | TMT翻译 | 简体中文 | `zh-CN` |
| `en` 英语 | `traditional` | 英语 | TMT翻译 | 繁体中文 | `zh-TW` |
| `en` 英语 | `en` | 英语 | 不转换 | 英语 | `en` |

### 患者说话 → 医生看到（由 `doctorOutputFormat` 控制）

规则同上，将说话语言替换为 `patientSpeakLanguage`，输出格式替换为 `doctorOutputFormat`。

## 七、注意事项

1. **音频格式**：音频流WebSocket只接受PCM格式（16kHz、16bit、单声道），前端需从TRTC SDK获取原始音频数据并转换。

2. **isFinal字段**：`isFinal=false` 为中间结果（正在说话，文本可能变化），`isFinal=true` 为最终结果（一句话说完，文本固定）。建议中间结果用打字机效果实时更新，最终结果固定显示。

3. **字幕显示**：前端应优先显示 `convertedText`（已转换/翻译后的文本），`originalText` 为ASR原始识别结果，可选择性显示。

4. **targetLanguage字段**：表示 `convertedText` 的语言，前端可据此判断字幕语言类型，用于UI标签显示（如"简体"、"繁体"、"English"）。

5. **WebSocket重连**：字幕WebSocket有4小时超时机制，长时间通话需处理重连逻辑。连接断开后需重新建立WebSocket连接。

6. **翻译延迟**：跨语言翻译（使用腾讯TMT）会有额外延迟（约100-300ms），同语言简繁转换（opencc4j）几乎无延迟。

7. **降级策略**：如果TMT翻译服务不可用，系统会自动降级返回原始ASR识别文本，不会导致字幕中断。

8. **userId匹配**：字幕WebSocket的 `userId` 路径参数需要与开始录制时使用的userId一致，否则无法收到字幕推送。医生端为 `doctor_{doctorId}`，患者端为手机号。
